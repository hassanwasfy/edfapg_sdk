iOS Delivery

XCFramework binary is resolved via Swift Package Manager from the public package repository.

Release Model

Each version publishes Android packages, iOS binary zip, and an updated Package.swift descriptor.

Android Usage Guide

Use this setup in host Android projects (Gradle Kotlin DSL).

1) Add repositories
dependencyResolutionManagement {
    repositories {
        maven("https://hassanwasfy.github.io/edfapg_sdk/maven")
        google()
        mavenCentral()
    }
}
2) Add dependency
dependencies {
    implementation("com.edfapay:edfapg_sdk:0.0.1")
}
3) Optional local testing
dependencyResolutionManagement {
    repositories {
        mavenLocal()
        google()
        mavenCentral()
    }
}

iOS Usage Guide (SwiftPM)

Use Xcode package dependencies with the public repository URL.

Package URL
https://github.com/hassanwasfy/edfapg_sdk
Xcode steps
1. File -> Add Package Dependencies...
2. Enter: https://github.com/hassanwasfy/edfapg_sdk
3. Select an exact version (for example: 0.0.1)
4. Add product: edfapg_sdk to your app target
Swift import
import edfapg_sdk

Troubleshooting

  • Remove old package paymentgateway_sdk from iOS hosts before adding edfapg_sdk.
  • In Xcode: reset package caches and resolve package versions again.
  • Ensure Android hosts use the exact GitHub Pages Maven URL listed above.
  • Use published versions only (do not request a version before release is completed).
